# 3.2 Property Access Dot vs Bracket

Phase: JS Reactivation
Topic: Objects
Status: Expanded

## Fast Goal
Know when dot notation works and when bracket notation is required.

## Brain Hook
Dot is direct. Bracket is dynamic.

## Why This Matters

You read object properties constantly in real code.
If the key is dynamic and you use the wrong access style, you get the wrong value or `undefined`.

## Micro Rule

Use dot for fixed property names and bracket for dynamic keys.

## Core Idea

Objects support two common ways to access properties.

Dot notation uses a known property name directly.

```ts
user.name
```

Bracket notation uses a string or variable.

```ts
user["name"]
user[key]
```

If the property name comes from a variable, bracket notation is required.

## Code Examples

Good:

```ts
const user = {
	name: "Ava",
	city: "Cebu",
};

const key = "city";

console.log(user.name);
console.log(user[key]);
```

Bad:

```ts
const user = {
	name: "Ava",
	city: "Cebu",
};

const key = "city";

console.log(user.key);
```

`user.key` looks for a property literally named `key`.

## Fast Comparison

Dot notation is best for fixed property names.
Bracket notation is required for dynamic keys and variable-based access.

## Mental Model

Dot is pointing straight at a label.
Bracket is checking a note first to see which label to use.

## Tiny Example

```ts
const product = {
	title: "Notebook",
};

console.log(product.title);
console.log(product["title"]);
```

## Output Prediction Drill

What is the output?

```ts
const profile = {

};

console.log(profile.role);
```


## Common Mistakes

- Using dot notation with a variable name
- Forgetting brackets need a string or variable
- Thinking `obj.key` and `obj[key]` always mean the same thing
- Misspelling the dynamic key

## Debug Check

What is wrong here?

```ts
const user = {
	name: "Ava",
	city: "Cebu",
};

const key = "city";

console.log(user.key);
```

Answer:

It reads a property literally named `key`.
To use the variable value, write `user[key]`.

## Meeting Answer

Dot notation is for direct property names, and bracket notation is for dynamic keys.

## Perfect Explanation

Use dot notation when the property name is fixed.
Use bracket notation when the key comes from a variable or expression.

## Guiding Questions

- Is the property name fixed or dynamic?
- Am I using the word `key` or the value inside `key`?
- Which notation matches the way the property name is being provided?
