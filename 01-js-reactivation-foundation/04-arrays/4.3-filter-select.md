# 4.3 Filter (Select)

Phase: JS Reactivation
Topic: Arrays
Status: Expanded

## Fast Goal
- Use `filter()` to produce a new array containing only items that match a predicate.

## Brain Hook
- Filter removes items; it narrows the list.

## Why This Matters
- You often need to pick relevant data from API responses or user input (active items, valid entries, search results).
- `filter` keeps code declarative and focused on selection logic.

## Micro Rule
- Use `filter` when you need a subset of the original array based on a boolean test.

## Core Idea
- `filter(predicate)` runs `predicate(item, index)` for each element and returns a new array of items where predicate returns `true`.

## Code Examples
- Good:
```ts
const nums = [1,2,3,4];
const evens = nums.filter(n => n % 2 === 0); // [2,4]
```

- Bad (common mistake):
```ts
const xs = [1,2,3];
xs.filter(n => { if(n>1) n }); // returns [] because block body returns undefined
```

## Fast Comparison
- `filter` vs `map`: `filter` changes the count (removes items); `map` keeps the count but changes each item.

## Mental Model
- Think of `filter` as a sieve: items pass through only if they meet the condition.

## Tiny Example
- `[1,2,3].filter(n => n>1)` => `[2,3]`

## Output Prediction Drill
- What does this return?
```ts
const users = [{active:true},{active:false},{active:true}];
console.log(users.filter(u => u.active));
```
- Answer: `[{active:true},{active:true}]`

## Your Task
- Implement `keepAdults(people)` that filters an array of `{name,age}` to only ages >= 18.

## Expected Solution
- Example:
```ts
function keepAdults(people: {name:string;age:number}[]) {
	return people.filter(p => p.age >= 18);
}
```

## Common Mistakes
- Using `map` instead of `filter`.
- Returning non-boolean values from predicate.
- Mutating the original array during filtering.

## Debug Check
- Broken code:
```ts
const a = [1,2,3];
const r = a.filter(x => { x > 1 });
console.log(r);
```
- What's wrong? Answer: block body lacks `return`, so predicate returns `undefined` (treated as `false`).

## Meeting Answer
- "`filter` runs a boolean test for each item and returns a new array with only the items that passed."

## Perfect Explanation
- `filter` keeps elements where the predicate returns `true`, producing a smaller (or equal) array.
