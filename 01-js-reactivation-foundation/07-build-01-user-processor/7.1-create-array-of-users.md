# 7.1 Create Array of Users

Phase: JS Reactivation
Topic: Build #1 - User Processor
Status: Expanded

## Fast Goal
- Create a realistic input array of user objects to drive the pipeline.

## Brain Hook
- Every pipeline is only as good as its input shape.

## Why This Matters
- A consistent, well-shaped input lets you write simple, predictable transformations.
- In production, clear shapes reduce bugs and make filters/transforms easier to test.

## Micro Rule
- Choose minimal fields you need for downstream tasks; avoid noisy extra properties.

## Core Idea
- Define a short array of user objects with consistent keys (id, name, age, email, active) so later steps can filter and map reliably.

## Code Examples
- Good:
```ts
const users = [
	{ id: 1, name: 'Asha', age: 28, email: 'asha@example.com', active: true },
	{ id: 2, name: 'Ben', age: 17, email: 'ben@example.com', active: false }
]
```

- Bad (inconsistent shape):
```ts
const users = [
	{ uid: 1, fullName: 'Asha' },
	{ id: 2, name: 'Ben', dob: '2008-01-01' }
]
```

## Fast Comparison
- Use a small normalized shape for processing vs a larger verbose shape for storage; processing shape favors consistency.

## Mental Model
- Think of the array as a table row set — every row must have the same columns for filters and maps to work.

## Tiny Example
```ts
const users = [{ id:1, name:'A', age:30 }]
```

## Output Prediction Drill
What is the length and first element's `name`?
```ts
const users = [{ id:1, name:'A', age:30 }, { id:2, name:'B', age:25 }]
console.log(users.length, users[0].name)
```
Answer: prints `2 'A'`.

## Your Task
- Create a `users` array (5 items) following the processing shape and include realistic sample values.

## Expected Solution
- An array of 5 objects with keys: `id` (number), `name` (string), `age` (number), `email` (string), `active` (boolean).

## Common Mistakes
- Mixing key names between entries (`uid` vs `id`).
- Storing age as string instead of number.
- Missing `active` flag when downstream logic expects it.

## Debug Check
What is wrong?
```ts
const users = [{ id: '1', name: 'A' }]
// later: users.filter(u => u.age >= 18)
```
Answer: `id` is a string and `age` is missing; `u.age >= 18` will be `undefined >= 18` -> false or runtime confusion.

## Perfect Explanation
