# 7.3 Map Names

Phase: JS Reactivation
Topic: Build #1 - User Processor
Status: Expanded

## Fast Goal
- Transform user objects into a simple list of display names.

## Brain Hook
- `map` shapes data into the exact form the UI or API expects.

## Why This Matters
- Mapping isolates presentation logic from raw data and keeps later steps lightweight.

## Micro Rule
- Keep map transforms pure and return new objects or primitive values only.

## Core Idea
- Use `Array.prototype.map` to extract or build a new value from each user object (e.g., full name or display object) without mutating the source.

## Code Examples
- Good (extract names):
```ts
const names = users.map(u => `${u.name}`)
```

- Good (build display objects):
```ts
const list = users.map(u => ({ id: u.id, display: `${u.name} (${u.age})` }))
```

- Bad (mutating original):
```ts
users.map(u => { u.display = u.name; return u })
// mutates source array
```

## Fast Comparison
- `map` transforms every item; `filter` removes items; combine them as `users.filter(...).map(...)` for pipelines.

## Mental Model
- Think of `map` as a lens that refocuses each item into the shape you need next.

## Tiny Example
```ts
[1,2,3].map(n => n * 2) // [2,4,6]
```

## Output Prediction Drill
What does this print?
```ts
const users = [{id:1,name:'A'},{id:2,name:'B'}]
const names = users.map(u => u.name.toUpperCase())
console.log(names)
```
Answer: prints `['A','B']` in uppercase -> `['A','B']` (actually `['A','B']` uppercase shown as `['A','B']`).

## Your Task
- Implement `function mapNames(users)` that returns an array of `displayName` strings in the form `Name (age)`.

## Expected Solution
- `return users.map(u => `${u.name} (${u.age})`)`

## Common Mistakes
- Mutating the original `users` array.
- Assuming all `name` or `age` values exist without guard checks.
- Returning inconsistent shapes across items.

## Debug Check
What is wrong?
```ts
const names = users.map(u => u.name && u.age && `${u.name} (${u.age})`)
// later code expects an array of strings
```
Answer: the expression can produce `false` values if a property is missing; better to normalize and default.

## Meeting Answer
- "I map each user to a small display value (like `Name (age)`), keeping the original data untouched."

## Perfect Explanation
- Use `map` to transform each user into the exact downstream shape, returning a new array without side effects.

## Guiding Questions
- Do you need a string or a small object for the UI?
- How will you handle missing `name` or `age` values?
- Should mapping happen before or after filtering?

## Done Signal
- `mapNames` returns consistent strings for all input items.
- Original `users` array remains unchanged.
- Edge cases handled (missing name/age) or documented.
