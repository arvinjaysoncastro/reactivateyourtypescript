# 4.1 Array Basics

Phase: JS Reactivation
Topic: Arrays
Status: Expanded

## Fast Goal
- Create, read, and update arrays fluently.

## Brain Hook
- Arrays are ordered buckets you can index, loop, and transform.

## Why This Matters
- Arrays are the primary container for collections in real apps.
- Most UI lists and fetched data are arrays; knowing safe, non-mutating transforms reduces bugs.

## Micro Rule
- Use zero-based indexes; prefer non-mutating methods (`map`, `filter`, `slice`) for transforms.

## Core Idea
- An array holds values in order. Access items by index, update by creating a new array when needed, and use array methods to transform data predictably.

## Code Examples
- Good example:

```ts
const names = ['aria', 'ben', 'carl'];
const first = names[0]; // 'aria'
const upper = names.map(n => n.toUpperCase());
```

- Bad example (shared reference):

```ts
const users = [{id:1, name:'a'}];
const copy = users; // both variables reference same array
```

## Fast Comparison
- `map` returns a new array (use for transformations). `forEach` runs side effects and returns `undefined`.

## Mental Model
- Picture an array as a train: cars in order labeled 0..n-1; you can swap cars or create a new train with modified cars.

## Tiny Example

```ts
const nums = [1,2,3];
nums[1] = 20; // [1,20,3]
```

## Output Prediction Drill
- What is the output?

```ts
const a = [1,2,3];

```

- Answer: `[1,2,3] [2,4,6]`

## Common Mistakes
- Using `splice` which mutates the original array.
- Confusing `map` with `forEach` for transformations.
- Off-by-one when slicing indexes.

## Debug Check
- What's wrong?

```ts
const arr = [1,2,3];
const copy = arr; 
copy.push(4);
// unexpected: arr also changed
```

- Answer: `copy` is a reference to the same array; use `arr.slice()` or `[...arr]` to copy.

## Meeting Answer
- Arrays are ordered collections accessed by zero-based indexes; use non-mutating transforms to avoid shared-reference bugs.

## Perfect Explanation
- An array stores values in order (index 0..n-1); prefer `map/filter/slice` to produce new arrays rather than mutating originals.
