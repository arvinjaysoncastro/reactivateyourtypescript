# 3.4 Object Mutation

Phase: JS Reactivation
Topic: Objects
Status: Expanded

## Fast Goal
See clearly when an object is being changed in place.

## Brain Hook
Mutation changes the original object, not a copy.

## Why This Matters

Objects are often shared across parts of a program.
If you mutate one object in place, other code using that same object can change unexpectedly.

## Micro Rule

If you change a property on the same object, that is mutation.

## Core Idea

Object mutation means changing the original object directly.
The object stays the same object, but one of its properties gets a new value.

```ts
const user = {
	name: "Ava",
	active: false,
};

user.active = true;
```

That updates the original object in place.
If another variable points to the same object, it will see that change too.

## Code Examples

Good for seeing mutation clearly:

```ts
const settings = {
	theme: "light",
};

settings.theme = "dark";
console.log(settings.theme);
```

Shared reference risk:

```ts
const user1 = {
	name: "Mina",
};

const user2 = user1;
user2.name = "Lia";

console.log(user1.name);
```

`user1.name` also changes because both variables point to the same object.

## Fast Comparison

Mutation changes the original object.
A copied update creates a new object with the change instead.

## Mental Model

Mutation is editing the same paper.
A copied update is making a new paper with the revised value.

## Tiny Example

```ts
const product = {
	price: 10,
};

product.price = 12;
console.log(product.price);
```

## Output Prediction Drill

What is the output?

```ts
const profile = {
	city: "Cebu",
};

const nextProfile = profile;
nextProfile.city = "Manila";

console.log(profile.city);
console.log(nextProfile.city);
```

Answer:

```ts
"Manila"
"Manila"
```

## Expected Solution

```ts

	owner: "Ava",
	balance: 100,
	active: false,
};

account.balance = 150;
account.active = true;

console.log(account);

// The original account object was changed in place.
```

## Common Mistakes

- Thinking property mutation creates a new object automatically
- Forgetting that another variable may share the same object reference
- Confusing mutation with reassignment
- Not noticing that the original value was changed directly

## Debug Check

What is wrong with this reasoning?

```ts
const a = { count: 1 };
const b = a;
b.count = 2;

// Only b changed.
```

Answer:

That reasoning is wrong because `a` and `b` point to the same object, so the mutation affects both.

## Meeting Answer

Object mutation means changing the original object directly, which can affect every reference to that object.

## Perfect Explanation

Mutation changes an existing object in place.
If multiple variables share that object, they all see the updated value.

## Guiding Questions

- Am I changing the same object or creating a new one?
- Could another variable be pointing to this same object?
- Which exact property was mutated?

## Done Signal

- You can spot when an object is mutated in place
- You can explain why shared references make mutation risky
- You can mutate one property intentionally and describe the result
- You can distinguish mutation from creating a copied update
