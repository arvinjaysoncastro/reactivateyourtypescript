# 2.2 Arrow Functions

Phase: JS Reactivation
Topic: Functions
Status: Expanded

## Fast Goal
Use arrow functions naturally in modern JavaScript.

## Brain Hook
Arrow functions are compact functions built for inline flow.

## Why This Matters
You will see arrow functions constantly in modern JavaScript codebases, especially in array methods, callbacks, and frontend code.
If you hesitate on the syntax, you slow down on everyday work.

## Micro Rule
Use arrow functions when you want a short function expression, especially for callbacks and small transforms.

## Core Idea
An arrow function is a shorter way to write a function expression.
It is especially useful when passing a function into another function, like `map`, `filter`, or `setTimeout`.

## Code Examples

Good:

```js
const double = (value) => value * 2;
const names = users.map((user) => user.name);
```

Less useful:

```js
const double = function (value) {
	return value * 2;
};
```

Risky for object methods:

```js
const user = {
	name: "Ava",
	sayName: () => {
		console.log(this.name);
	},
};
```

Better for object methods when you need the object context:

```js
const user = {
	name: "Ava",
	sayName() {
		console.log(this.name);
	},
};
```

## Fast Comparison
- Function declaration: named, hoisted, good for main reusable logic
- Arrow function: compact, commonly stored in variables, great for callbacks and transforms

## Mental Model
Function declarations are like full written instructions.
Arrow functions are like short sticky notes you pass into another action.

## Tiny Example

```js
const greet = (name) => {
	console.log("Hello " + name);
};

greet("Mina");
```

## Your Task
You have this data:

```js
const tasks = [
	{ title: "Email client", done: true },
	{ title: "Write report", done: false },
	{ title: "Fix bug", done: true },
];
```

Create arrow functions that:
1. keep only done tasks
2. transform the result into an array of titles
3. print the final array

## Expected Solution

```js
const tasks = [
	{ title: "Email client", done: true },
	{ title: "Write report", done: false },
	{ title: "Fix bug", done: true },
];

const doneTasks = tasks.filter((task) => task.done);
const doneTitles = doneTasks.map((task) => task.title);

console.log(doneTitles);
```

## Common Mistakes
- forgetting parentheses when using multiple parameters
- using braces but forgetting `return`
- thinking arrow functions and function declarations are identical in all situations
- using arrow functions for object methods when `this` matters

## Debug Check

```js
const numbers = [1, 2, 3];
const doubled = numbers.map((num) => {
	num * 2;
});

console.log(doubled);
```

What is wrong?
Answer: braces were used without `return`, so each result becomes `undefined`.

## Meeting Answer
Arrow functions are shorter function expressions that are especially useful for callbacks and quick transforms.

## Perfect Explanation
An arrow function is a compact way to write a function expression.
It is most useful for short logic, especially inside callbacks like `map`, `filter`, and `find`.
