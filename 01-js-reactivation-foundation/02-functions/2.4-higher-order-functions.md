# 2.4 Higher-Order Functions

Phase: JS Reactivation
Topic: Functions
Status: Expanded

## Fast Goal
Pass functions into functions without panic.

## Brain Hook
A higher-order function uses functions as data.

## Why This Matters
Modern JavaScript uses higher-order functions everywhere: array methods, event handlers, timers, and reusable utilities.
If you understand callbacks clearly, a lot of JavaScript stops looking magical.

## Micro Rule
If a function takes another function or returns another function, it is a higher-order function.

## Core Idea
A higher-order function works with other functions.
Usually this means one function receives a callback and decides when to run it, or returns a new function for later use.

## Code Examples

Good:

```js
function runTask(taskName, callback) {
	console.log("Starting: " + taskName);
	callback();
}

runTask("Email", () => {
	console.log("Task finished");
});
```

Good return-value example:

```js
function makeGreeter(name) {
	return function () {
		console.log("Hello " + name);
	};
}
```

Bad mental model:

```js
function runTask(taskName, callback) {
	console.log("Starting: " + taskName);
	console.log(callback);
}

___

function runTask(task: () => void): void { task(); }

function sayHello(): void { console.log("Hello"); }

// Wrong should be runTask(sayHello)
runTask(sayHello());
```

This only prints the function instead of running it.

## Fast Comparison
- Normal function: receives values and returns values
- Higher-order function: receives functions or returns functions

## Mental Model
A higher-order function is like a manager giving instructions to another worker.
The manager decides when that worker starts.

## Tiny Example

```js
function sayHello() {
	console.log("Hello");
}

function runLater(action) {
	action();
}

runLater(sayHello);
```

## Output Prediction Drill

```js
function runTwice(action) {
	action();
	action();
}

runTwice(() => console.log("Go"));
```

Answer:

```txt
Go
Go
```

## Your Task
Create a function called `processUsers`.

Requirements:
1. It must take two parameters: an array of user names and a callback.
2. It must loop through the names.
3. It must call the callback once for each name.
4. Then call `processUsers` with a callback that prints `User: ` plus the name.

## Expected Solution

```js
function processUsers(names, callback) {
	for (const name of names) {
		callback(name);
	}
}

processUsers(["Ava", "Noah", "Mina"], (name) => {
	console.log("User: " + name);
});
```

## Common Mistakes
- calling the callback too early with `callback()` when you meant to pass the function itself
- forgetting to give the callback the value it needs
- thinking only `map` and `filter` are higher-order functions
- printing the callback instead of running it

## Debug Check

```js
function runTask(callback) {
	callback;
}

runTask(() => console.log("Done"));
```

What is wrong?
Answer: the callback is never executed because `callback;` only references it. It should be `callback()`.

## Meeting Answer
A higher-order function uses another function as input or output.

## Perfect Explanation
A higher-order function either takes a function as an argument or returns a function.
Callbacks inside `map`, `filter`, timers, and custom utilities are common examples.

## Done Signal
- You can pass a callback into a function without calling it too early.
- You can explain why `map` and `filter` are higher-order functions.
- You can write a function that runs a callback for each item in an array.
- You can clearly describe the difference between passing a function and calling a function.
