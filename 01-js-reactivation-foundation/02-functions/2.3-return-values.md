# 2.3 Return Values

Phase: JS Reactivation
Topic: Functions
Status: Ready for expansion

## Fast Goal
Understand what a function gives back and why missing `return` causes bugs.

## Brain Hook
No `return` means no useful output.

## Why This Matters
In real code, functions often need to produce a value that another part of the program will use. If a function forgets to return, the next step gets `undefined`, and the bug spreads.

## Micro Rule
`return` sends a value out of a function.

## Core Idea
A function can do work inside its body, but only `return` gives a value back to the caller.

`console.log()` only shows something on the screen.  
`return` gives something back so other code can use it.

## Code Examples

### Good
```ts
function add(a: number, b: number): number {
  return a + b;
}

const total = add(2, 3);
console.log(total); // 5
````

### Bad

```ts
function add(a: number, b: number): void {
  console.log(a + b);
}

const total = add(2, 3);
console.log(total); // undefined
```

## Fast Comparison

### `return`

* sends value back
* can be stored
* can be reused later

### `console.log`

* only prints value
* cannot be used as output
* mostly for debugging

## Mental Model

A function is like a vending machine.

You put something in.
`return` is the snack coming out.
No `return` means nothing comes out.

## Tiny Example

```ts
function getName(): string {
  return "Arvin";
}

const name = getName();
console.log(name); // Arvin
```

## Expected Solution

```ts

  return "Welcome, " + name;
}

const message = getWelcomeMessage("Arvin");
console.log(message);
```

## Common Mistakes

* using `console.log()` instead of `return`
* forgetting to return a value
* expecting a function to output usable data without `return`
* writing code after `return` and expecting it to run

## Debug Check

What is wrong here?

```ts
function getTotal(a: number, b: number): number {
  const sum = a + b;
}

const finalTotal = getTotal(4, 6);
console.log(finalTotal);
```

## Meeting Answer

A return value is the value a function sends back to the caller. Without `return`, the function gives back `undefined`, so other code cannot use the result properly.

## Perfect Explanation

`return` gives a value back from a function.
Without it, the result is usually `undefined`.

## Guiding Questions

* What comes back out of the function?
* Can another variable use this result?
* Are you printing the value, or actually returning it?
