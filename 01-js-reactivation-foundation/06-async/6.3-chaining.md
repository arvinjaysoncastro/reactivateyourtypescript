# 6.3 Chaining

Phase: JS Reactivation
Topic: Async
Status: Expanded

## Fast Goal
- Chain promise steps so each result flows to the next without losing order.

## Brain Hook
- Chaining passes each result into the next step — like an assembly line for async results.

## Why This Matters
- Many APIs require multi-step async flows (fetch → parse → transform). Promises chaining keeps steps readable and predictable when used correctly.

## Micro Rule
- Return a promise from each `then` callback when performing async work; otherwise the chain breaks or runs out of sequence.

## Core Idea
- Promise chaining links handlers: `.then(result => next(result))` returns the next promise so the chain waits for it before continuing.

## Code Examples
- Proper chaining:
```ts
fetch(url)
	.then(r => r.json())
	.then(data => process(data))
	.then(result => console.log(result))
	.catch(err => console.error(err));
```

- Broken chaining (common mistake):
```ts
fetch(url)
	.then(r => r.json())
	.then(data => { doAsync(data); }) // forgot to return the inner promise
	.then(() => console.log('done'));
```

## Fast Comparison
- `then` chaining vs `async/await`: `await` flattens chains into sequential code; chaining makes the pipeline explicit and composable.

## Mental Model
- Picture a promise chain as a pipeline where each stage receives the previous stage's output and can return a new promise for the next stage.

## Tiny Example
- `Promise.resolve(1).then(x => x+1).then(x => x*2)` => resolves to `4`.

## Output Prediction Drill
- What resolves here?
```ts
Promise.resolve(2)
	.then(x => x + 3)
	.then(x => Promise.resolve(x * 2))
	.then(x => console.log(x));
```
- Answer: prints `10` (2+3=5, then 5*2=10).

## Your Task
- Implement `chainAndSum(promisesFns)` where `promisesFns` is an array of functions that return `Promise<number>`; execute them in sequence and return a promise resolving to the sum of results.

## Expected Solution
- Example approach:
```ts
function chainAndSum(fns){
	return fns.reduce((p, fn) => p.then(sum => fn().then(v => sum + v)), Promise.resolve(0));
}
```

## Common Mistakes
- Not returning inner promises from `then` callbacks.
- Mixing `then` and `await` incorrectly without understanding flow.
- Using `forEach` for sequential async calls — it does not await.

## Debug Check
- Broken code:
```ts
const arr = [() => Promise.resolve(1), () => Promise.resolve(2)];
arr.forEach(fn => fn().then(console.log)); // runs in parallel, not sequential
```
- What's wrong? Answer: `forEach` doesn't chain or await; use chaining or `for..of` with `await`.

## Meeting Answer
- "Chaining connects promise-returning steps by returning the next promise from each `then`, preserving sequence and flow."

## Perfect Explanation
- In a chain, each `then` returns a value or promise; returning a promise makes the chain wait for it before continuing.
