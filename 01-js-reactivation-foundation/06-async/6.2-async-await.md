# 6.2 Async Await

Phase: JS Reactivation
Topic: Async
Status: Expanded

## Fast Goal
- Use `async`/`await` to write clear, top-to-bottom asynchronous code.

## Brain Hook
- `await` pauses the async function, not the whole program.

## Why This Matters
- `async`/`await` makes asynchronous flows readable and maintainable; it's how modern code handles network calls, file I/O, and timed events.

## Micro Rule
- Use `await` only inside an `async` function; always handle rejections with `try/catch` or by returning the promise.

## Core Idea
- `async` marks a function that returns a `Promise`. Inside it, `await promise` suspends execution until the promise settles and yields the resolved value.

## Code Examples
- Promise chain converted to `await`:
```ts
// then-chain
fetchData().then(r => r.json()).then(data => console.log(data));

// async/await
async function load(){
	const res = await fetchData();
	const data = await res.json();
	console.log(data);
}
```

- Bad (missing try/catch):
```ts
async function bad(){
	const v = await mayReject(); // unhandled rejection if mayReject throws
}
```

## Fast Comparison
- `then` chains vs `await`: `await` reads like sync code and is easier to reason about for sequential steps.

## Mental Model
- `await` is like a bookmark inside a function: the rest of the function waits there until the result is ready.

## Tiny Example
- `async function f(){ return await Promise.resolve(3) }` returns a `Promise` resolved to `3`.

## Output Prediction Drill
- What prints in which order?
```ts
console.log('start');
(async ()=>{ await Promise.resolve(); console.log('inside'); })();
console.log('end');
```
- Answer: `start`, `end`, `inside` — awaited microtasks run after current tick.

## Common Mistakes
- Using `await` outside `async` functions.
- Forgetting to `await` when needed and treating the promise as the value.
- Not handling rejections leading to uncaught errors.

## Debug Check
- Broken code:
```ts
async function f(){
	const a = mayReject();
	const b = await a; // forgot to handle mayReject with try/catch
}
```
- What's wrong? Answer: `mayReject()` returns a promise that might reject; use `try/catch` around `await a` to handle rejections.

## Meeting Answer
- "`async` functions return promises; `await` pauses the function until the promise resolves — always handle errors."

## Perfect Explanation
- `async`/`await` makes promise-based code look synchronous by pausing function execution at `await` until the promise settles.
