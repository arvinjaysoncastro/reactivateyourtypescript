# 21.3 Single Responsibility

Phase: Architect Structure
Topic: Clean Code
Status: Ready for expansion

## Fast Goal
Keep one unit of code focused on one reason to change.

## Brain Hook
If code does many jobs, it becomes hard to trust and hard to change.

## Session Flow
- inspect overloaded code
- identify multiple jobs
- separate responsibilities
- explain the improvement
