# 3.3 Nested Objects

Phase: JS Reactivation
Topic: Objects
Status: Expanded

## Fast Goal
Read and update nested data without getting lost.

## Brain Hook
Nested objects are just objects inside objects.

## Why This Matters

Real app data is often layered.
If you can follow and update nested object paths clearly, you avoid reading the wrong value or changing the wrong field.

## Micro Rule

Read nested data one level at a time and follow the full property path exactly.

## Core Idea

A nested object is an object stored inside another object.
You access deep values by following the property path step by step.

```ts
const user = {
	name: "Ava",
	address: {
		city: "Cebu",
		zip: "6000",
	},
};
```

To read `city`, you follow the path:

```ts
user.address.city
```

To update it, target the same path directly.

## Code Examples

Good:

```ts
const profile = {
	name: "Mina",
	contact: {
		email: "mina@example.com",
	},
};

console.log(profile.contact.email);
profile.contact.email = "mina.dev@example.com";
```

Broken path:

```ts
const profile = {
	contact: {
		email: "mina@example.com",
	},
};

console.log(profile.email);
```

`profile.email` is wrong because `email` is inside `contact`.

## Fast Comparison

Flat object access uses one property name.
Nested object access uses a full path through each level.

## Mental Model

Think of a nested object like folders inside folders.
To reach the file you want, you open each folder in order.

## Tiny Example

```ts
const settings = {
	ui: {
		theme: "dark",
	},
};

console.log(settings.ui.theme);
```

## Common Mistakes

- Forgetting one level in the property path
- Trying to read a deep value from the top level
- Updating the wrong nested field
- Getting lost because the object shape was not read first

## Debug Check

What is wrong here?

```ts
const user = {
	contact: {
		email: "ava@example.com",
	},
};

console.log(user.email);
```

Answer:

`email` is nested inside `contact`, so the correct path is `user.contact.email`.

## Meeting Answer

Nested objects are objects inside other objects, and you read them by following the full property path.

## Perfect Explanation

A nested object stores one object inside another.
To access a deep value, move through each level in order using dot notation.

- Which part is top-level and which part is nested?
- Are you updating the exact nested field or the wrong level?
