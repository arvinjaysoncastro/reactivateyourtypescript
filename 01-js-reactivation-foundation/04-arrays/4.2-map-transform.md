# 4.2 Map (Transform)

Phase: JS Reactivation
Topic: Arrays
Status: Expanded

## Fast Goal
- Turn one array shape into another with `map()`.

## Brain Hook
- Map keeps the count; it replaces each item.

## Why This Matters
- Real code often reshapes data from APIs to component props or view models.
- `map` is the efficient, declarative way to transform every element in an array.

## Micro Rule
- Use `map` when you need a new array with the same length but different items.

## Core Idea
- `map(callback)` runs `callback(item, index)` for each element and returns a new array built from the callback results.

## Code Examples
- Good:
```ts
const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2); // [2,4,6]
```

- Bad (common mistake):
```ts
const arr = [1,2,3];
const attempt = arr.forEach(n => n * 2); // undefined — forEach doesn't return a new array
```

## Fast Comparison
- `map` vs `forEach`: `map` returns a new array; `forEach` returns `void` and is for side effects.

## Mental Model
- Think of `map` as a conveyor belt: items go in, transformation happens, same number of items come out.

## Tiny Example
- `['a','b'].map(s => s.toUpperCase())` => `['A','B']`

## Output Prediction Drill

## Your Task
- Implement `formatUsers(users)` that maps an array of `{id,name,email}` to an array of greeting strings: `"Hi <name> — <email>"`.

## Expected Solution
- Example implementation:
```ts
function formatUsers(users: {id:number;name:string;email:string}[]) {
	return users.map(u => `Hi ${u.name} — ${u.email}`);
}
```

## Common Mistakes
- Using `forEach` instead of `map` and expecting a return value.
- Mutating the original array inside the callback.
- Returning `undefined` from the callback.

## Debug Check
- Broken code:
```ts
const xs = [1,2,3];
const r = xs.map(x => { x * 2 });
console.log(r);
```
- What's wrong? Answer: callback uses block body but doesn't `return` — results are `[undefined,undefined,undefined]`.

## Meeting Answer
- "`map` transforms each element and returns a new array of the same length with transformed items."

## Perfect Explanation
- `map` runs a transform on each item and returns a new array built from those returns.
