# 4.4 Reduce (Combine)

Phase: JS Reactivation
Topic: Arrays
Status: Expanded

## Fast Goal
- Use `reduce()` to combine array items into a single result without confusion.

## Brain Hook
- Reduce carries one running result across every step.

## Why This Matters
- Many real tasks need a single summary from a list: sums, aggregates, grouped objects, or derived state for UI.
- `reduce` expresses these clearly and avoids manual bookkeeping with loops.

## Micro Rule
- Use `reduce` when you need one final value derived from all items (number, object, string, etc.).

## Core Idea
- `arr.reduce((acc, item, index) => { /* update acc */}, initial)` walks the array once and returns the accumulator after processing every item.

## Code Examples
- Sum numbers:
```ts
const nums = [1,2,3];
const sum = nums.reduce((acc, n) => acc + n, 0); // 6
```

- Build an index:
```ts
const users = [{id:1,name:'A'},{id:2,name:'B'}];
const byId = users.reduce((acc, u) => { acc[u.id]=u; return acc }, {} as Record<number,typeof users[0]>);
```

- Bad (common mistake):
```ts
const wrong = [1,2,3].reduce((acc,n) => { acc + n }); // forgot to return acc — results in unexpected value
```

## Fast Comparison
- `reduce` vs `map`/`filter`: `reduce` produces a single value; `map`/`filter` produce arrays.

## Mental Model
- Think of `reduce` as a running notebook: each item writes into the notebook and the final notebook is returned.

## Tiny Example
- `[1,2,3].reduce((s,n)=>s+n,0)` => `6`

## Output Prediction Drill

## Common Mistakes
- Forgetting to return the accumulator in the reducer.
- Using an incorrect initial value (e.g., `undefined` or wrong type).
- Mutating the accumulator in ways that break types or identity when using immutable patterns.

## Debug Check
- Broken code:
```ts
const s = [1,2].reduce((acc,n)=>{ acc += n });
```
- What's wrong? Answer: reducer uses block body but doesn't `return acc`, so result is undefined or incorrect.

## Meeting Answer
- "`reduce` walks items while updating an accumulator and returns the final accumulated value."

## Perfect Explanation
- `reduce` applies a reducer to each item, updating an accumulator, and returns the accumulator after the last item.
