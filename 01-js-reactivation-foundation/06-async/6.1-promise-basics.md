# 6.1 Promise Basics

Phase: JS Reactivation
Topic: Async
Status: Expanded

## Fast Goal
- Treat a `Promise` as a definite future value you can handle with handlers or `await`.

## Brain Hook
- A promise is a placeholder for a result that will arrive later — not a value now.

## Why This Matters
- Modern code frequently interacts with I/O and waits for results. Understanding promises prevents race conditions, unhandled rejections, and callback hell.

## Micro Rule
- Use `then`/`catch` or `async/await` to handle a promise's fulfilled or rejected outcome; never assume it is synchronous.

## Core Idea
- A `Promise` represents an operation that may complete in the future; it has states: `pending`, `fulfilled`, or `rejected`. Handlers run when the state settles.

## Code Examples
- Good (basic):
```ts
const p = new Promise<number>((resolve) => setTimeout(() => resolve(42), 50));
p.then(v => console.log(v));
```

- Good (async/await):
```ts
async function fetchNumber(){
	const v = await p; // pauses until p resolves
	return v * 2;
}
```

- Bad (common mistake):
```ts
const x = p; // x is a Promise, not the number — don't treat it like the resolved value
```

## Fast Comparison
- `then` vs `await`: `then` uses callbacks; `await` makes async code appear synchronous inside `async` functions.

## Mental Model
- Think of a promise as a ticket number: you hold it now and collect a meal later when your number is called.

## Tiny Example
- `Promise.resolve('hi').then(s => console.log(s.toUpperCase()))` prints `HI`.

## Output Prediction Drill
- What prints and why?
```ts
console.log('A');
Promise.resolve().then(()=>console.log('B'));
console.log('C');
```
- Answer: `A`, `C`, then `B` — microtask queue runs promise handlers after current tick.

## Common Mistakes
- Treating a promise as its resolved value.
- Forgetting to return a promise from an `async` function.
- Not handling rejections (`catch`) leading to unhandled promise rejections.

## Debug Check
- Broken code:
```ts
function f(){
	const p = new Promise((res,rej)=>{ throw new Error('boom') });
	p.then(v=>console.log(v));
}
f();
```
- What's wrong? Answer: the executor threw synchronously; without a `catch` the rejection is unhandled — add `p.catch(...)` or handle errors in `then` chain.

## Meeting Answer
- "A promise is a future value with three states; handle it with `then`/`catch` or `await` and avoid treating it as immediate."

## Perfect Explanation
- A `Promise` represents a future outcome; attach handlers or `await` to act when it settles.
