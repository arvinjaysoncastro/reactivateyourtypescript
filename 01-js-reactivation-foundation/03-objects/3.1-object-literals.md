# 3.1 Object Literals

Phase: JS Reactivation
Topic: Objects
Status: Expanded

## Fast Goal
Create small objects confidently and read their shape at a glance.

## Brain Hook
An object groups related data under named keys.

## Why This Matters

Real code often needs one value to describe one thing, like a user, task, product, or settings object.
If object literals feel natural, your data becomes easier to read, pass around, and update.

## Micro Rule

Use an object when several values belong to the same thing.

## Core Idea

An object literal is a value created with curly braces.
It stores related data as `key: value` pairs.

```ts
const user = {
	name: "Ava",
	age: 28,
	active: true,
};
```

You read properties with dot notation and update them by targeting the property directly.

## Code Examples

Good:

```ts
const product = {
	title: "Notebook",
	price: 12,
	inStock: true,
};

console.log(product.title);
product.price = 15;
```

Less clear for related data:

```ts
const title = "Notebook";
const price = 12;
const inStock = true;
```

## Fast Comparison

Separate variables work when values are unrelated.
An object is better when the values describe one thing.

## Mental Model

Think of an object as one labeled container.
Each property is a label with a value inside it.

## Tiny Example

```ts
const car = {
	brand: "Toyota",
	color: "Blue",
};

console.log(car.brand);
```

## Common Mistakes

- Forgetting commas between properties
- Using `=` instead of `:` inside the object
- Reading a property name that does not exist
- Splitting related values into separate variables
- Forgetting that dot notation targets one property

## Debug Check

What is wrong here?

```ts
const user = {
	name = "Ava",
	age: 30,
};
```

Answer:

Inside an object literal, properties use `:` not `=`.

## Meeting Answer

An object literal groups related data into one value using named properties.

## Perfect Explanation

An object literal creates one value with related key-value pairs.
You use dot notation to read or update its properties.
