# 7.2 Filter Adults

Phase: JS Reactivation
Topic: Build #1 - User Processor
Status: Expanded

## Fast Goal
- Write a precise predicate to select adult users from the dataset.

## Brain Hook
- Filters remove noise so the steps that follow see only the users you care about.

## Why This Matters
- Correct predicates prevent downstream bugs and ensure business rules (e.g., age limits) are applied reliably.

## Micro Rule
- Keep predicates simple and explicit: `u => typeof u.age === 'number' && u.age >= 18`.

## Core Idea
- Use `Array.prototype.filter` with a clear predicate to create a subset of users matching the adult rule; prefer explicit checks to avoid type surprises.

## Code Examples
- Good:
```ts
const adults = users.filter(u => typeof u.age === 'number' && u.age >= 18)
```

- Bad (loose check):
```ts
const adults = users.filter(u => u.age)
// fails: ages 0 or missing values break the logic
```

## Fast Comparison
- `filter` returns many items; `find` returns the first match; use `filter` when you need the full set.

## Mental Model
- Think of `filter` as a sieve with holes defined by the predicate — only items that fit through the holes remain.

## Tiny Example
```ts
[{id:1,age:20},{id:2,age:16}].filter(u => u.age >= 18)
```

## Output Prediction Drill
What is the result?
```ts
const users = [{id:1,age:18},{id:2,age:17},{id:3,age:'19'}]
const adults = users.filter(u => typeof u.age === 'number' && u.age >= 18)
console.log(adults)
```
Answer: prints `[{id:1,age:18}]` — the string `'19'` is excluded.

## Your Task
- Implement `function filterAdults(users)` that returns only users where `age` is a number >= 18.

## Expected Solution
- `return users.filter(u => typeof u.age === 'number' && u.age >= 18)`

## Common Mistakes
- Using `u => u.age` which fails for `0` or string ages.
- Forgetting to check for `undefined` or non-number ages.
- Mixing `find` and `filter` semantics.

## Debug Check
What is wrong?
```ts
const adults = users.filter(u => u.age >= 18)
// but some users have age: '20'
```
Answer: string ages will be coerced or cause unexpected behavior; prefer explicit `typeof` check.

## Meeting Answer
- "I filter the user array with a strict predicate that ensures `age` is numeric and at least 18, optionally requiring `active` where needed."

## Perfect Explanation
- Use `filter` with `typeof u.age === 'number' && u.age >= 18` to reliably select adult users.

## Guiding Questions
- Do all entries have `age` as a number? If not, how will you handle them?
- Should inactive users be excluded too?
- What edge ages do business rules require (e.g., 18 vs 21)?
